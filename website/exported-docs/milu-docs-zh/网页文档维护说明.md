# CoPaw 网页文档维护说明

本文档说明如何维护网页内展示的 CoPaw 中文文档（文本、图片与自动重载）。

## 文档来源与访问地址

- 文本源文件：`website/exported-docs/copaw-docs-zh/copaw-docs-zh-all.local.md`
- 图片目录：`website/exported-docs/copaw-docs-zh/images`
- 网页地址：`http://127.0.0.1:8088/copaw-docs-zh`

## 修改文本

1. 编辑 `copaw-docs-zh-all.local.md`。
2. 使用标准 Markdown 语法（标题、列表、代码块、表格、超链接等）。
3. 保存后，文档页会自动检测变更并刷新。

## 修改图片

1. 将图片放入 `images` 目录。
2. 在 Markdown 中使用相对路径引用，例如：
   - `![控制台示意](./images/img-001.png)`
3. 保存后，文档页会自动检测变更并刷新。

## 超链接写法

- 文档内锚点：`[跳到快速开始](#quickstart)`
- 外部链接：`[AgentScope](https://github.com/agentscope-ai)`
- 本地图片/文件：使用相对路径（会由后端自动映射到文档静态资源路由）

## 自动重载机制

- 页面每 3 秒请求一次 `/copaw-docs-zh-meta` 获取文档版本。
- 若检测到版本变化（Markdown 或 `images` 下文件变更），页面会自动刷新。

## 常见问题

- 图片不显示：先检查文件是否在 `images` 目录，路径大小写是否一致。
- 修改后未更新：确认服务仍在运行，再检查浏览器是否被扩展拦截自动刷新脚本。

## 顶栏 Logo（浅色 / 深色）

- 源文件放在：`console/public/logo.png`（浅色）、`console/public/dark-logo.png`（深色）。
- Vite 构建会把 `public` 根下文件拷到 `console/dist`，再同步到 `src/copaw/console` 后由 `copaw app` 从根路径提供 `/logo.png`、`/dark-logo.png`。
- 修改后请在 `console/` 目录执行 `npm run build`，并将 `console/dist` 同步到 `src/copaw/console`（与你们现有流程一致），然后重启 `copaw app`。
- 若浅色已更新、深色仍像旧图：多为浏览器缓存；服务端已对 Logo 响应使用 `Cache-Control: public, max-age=0, must-revalidate`；前端构建还会为 Logo URL 附带 `?v=构建时间戳`，每次 `npm run build` 后会换 URL。仍异常时请强制刷新（Ctrl+F5），并确认已替换 `dark-logo.png` 文件本身。
